# AstraDev End-to-End Test Plan

## Environment
- Frontend: http://localhost:3000
- Backend: http://localhost:8000
- All services confirmed running: Next.js, Daphne, Celery worker, PostgreSQL, Redis

## Test 1: User Signup and Authentication Flow

**Steps:**
1. Navigate to http://localhost:3000
2. Click "Get Started" link (goes to /signup)
3. Fill signup form: Display Name = "Test User", Email = "e2etest@astradev.io", Password = "TestPass123!"
4. Click "Create Account"

**Pass criteria:**
- After submit, browser redirects to `/dashboard`
- Dashboard page shows "My Projects" heading
- Dashboard shows "No projects yet" empty state
- Navigation shows "AstraDev" logo and "Logout" button

**Fail indicators:**
- Error banner appears with red text
- Stays on /signup page
- 4xx/5xx response in network

## Test 2: Create Project and Verify Agent Pipeline Triggers

**Steps:**
1. On Dashboard, click "+ New Project" button
2. Fill: Project name = "Hello World App", Prompt = "Build a simple Python hello world script that prints 'Hello AstraDev!'"
3. Leave "Build APK" unchecked
4. Click "Start Building"

**Pass criteria:**
- Browser redirects to `/projects/<uuid>` page
- Project page loads with three-panel layout
- Activity tab shows at least one message from agents (type: "thinking" or "action")
- Messages should appear within 30 seconds (Celery picks up task, Groq API responds)
- At minimum, a "thinking" message with "Analyzing request..." should appear

**Fail indicators:**
- Redirect doesn't happen (stays on dashboard)
- Project page shows no messages after 30+ seconds
- Error messages appear in activity stream
- 500 error on project creation API call

## Test 3: Verify Real-Time WebSocket Streaming

**Steps:**
1. On the project page from Test 2, observe the Activity tab
2. Wait up to 60 seconds for the agent pipeline to complete

**Pass criteria:**
- Multiple messages appear in the activity stream with different types (thinking, plan, action, code, success)
- Messages show agent attribution (e.g., "orchestrator", "planner", "writer")
- Final message should be a "success" type with content mentioning "completed"
- The messages are color-coded differently by type (visible in UI)

**Fail indicators:**
- Only one message or no messages appear
- All messages are "error" type
- WebSocket disconnects (no reconnection)

## Test 4: Verify Files Tab Shows Generated Code

**Steps:**
1. After agent pipeline completes (success message visible), click "Files" tab
2. Look for generated files in the file browser

**Pass criteria:**
- Files tab shows at least one file (e.g., a .py file or similar)
- Clicking a file shows its content with syntax highlighting
- The file content is relevant to the prompt (Python hello world)

**Fail indicators:**
- Files tab is empty even after pipeline completion
- Files show but content is empty or irrelevant

## Test 5: APK Build Functionality

**Steps:**
1. Go back to Dashboard
2. Create new project: Name = "APK Test App", Prompt = "Build a simple counter app", check "Build APK" checkbox
3. Click "Start Building"
4. On project page, click "Build APK" button in the top bar

**Pass criteria:**
- The "Build APK" button is visible in the project page top bar
- Clicking it either:
  - Returns success with download URL, OR
  - Returns "project_ready" status with message about downloading ZIP
- Response includes `files_generated` count > 0

**Fail indicators:**
- "Build APK" button not visible
- Button click produces a 500 error
- No response at all (hangs)

## Test 6: Download Workspace ZIP

**Steps:**
1. On the project page from Test 2 (after pipeline completes), click "Download ZIP" button

**Pass criteria:**
- A file download initiates
- Downloaded file is a valid ZIP archive
- ZIP contains files generated by the agents

**Fail indicators:**
- 404 error (no workspace files)
- Download button missing
- Empty ZIP file
